# Supervisor Packet Generation and Manual Model Switching

## Purpose
Formalize packet schema and the fewest-messages manual switching format. The Packet is the unit of work in Dopemux.

## Scope
- JSON Schema for Task Packets.
- Chunking strategy for manual handoffs.
- Runner-specific prompt wrapping.

## Non-negotiable invariants
1. **Self-Contained**: A packet MUST contain all information necessary to execute it (or references to available context).
2. **Human Readable**: The core instruction field MUST be readable by a human operator.
3. **Audit Trail**: Every packet must point to its parent packet (if any).

## Packet schema (minimal)
```json
{
  "id": "TP-20240213-001",
  "parent_id": null,
  "timestamp": "2024-02-13T12:00:00Z",
  "title": "Implement Auth",
  "objective": "Add JWT authentication to the API",
  "runner_id": "claude-3-opus",
  "context_files": ["src/api/auth.py", "config/secrets.yaml"],
  "steps": [
    {
      "id": "1",
      "type": "shell",
      "command": "pip install pyjwt"
    },
    {
      "id": "2",
      "type": "edit",
      "file": "src/api/auth.py",
      "instruction": "Implement verify_token function"
    },
    {
      "id": "3",
      "type": "verify",
      "command": "pytest tests/test_auth.py"
    }
  ],
  "stop_conditions": [
    "Test failure",
    "Dependency conflict"
  ]
}
```

## Chunking format per step (manual mode)
When running in "Manual" or "Review" mode, the packet is sliced into chunks.

**Example Slicing**:
- **CHUNK 1**: Context & Plan Verification. (Runner: Haiku)
- **CHUNK 2**: Implementation Steps. (Runner: Sonnet/Opus)
- **CHUNK 3**: Verification & Cleanup. (Runner: Haiku)

**File Envelope**:
- `packet_chunk_1.md` (Markdown wrapper for LLM consumption).
- `packet_chunk_2.md`...

## Runner-specific wrappers
Different runners need prompts formatted differently.
- **Claude Code**: Needs explicit XML tags and `<context>` blocks.
- **Codex**: Needs comments and `#` headers.
- **Copilot**: Needs natural language conversational style.

Packet Generator applies these templates at runtime.

## Resume strategy
- **Sentinel Files**: `current_packet/step_1.done`.
- **Exit Codes**:
  - `0`: Success, proceed.
  - `1`: Failure, stop.
  - `2`: **Manual Handoff Required** (e.g., "I need you to check this diff").

## Failure modes
- **Chunk Mismatch**: Chunk 2 assumes state from Chunk 1 that wasn't persisted.
- **Hallucinated Step**: LLM adds a step "4" that isn't in the JSON. (TaskX ignores it, as it follows the JSON spec).

## Acceptance criteria
1. **Schema Check**: Validate a generated packet against `packet_schema.json`.
2. **Chunking Check**: Split a 10-step packet into 3 chunks. Ensure all steps are covered exactly once.

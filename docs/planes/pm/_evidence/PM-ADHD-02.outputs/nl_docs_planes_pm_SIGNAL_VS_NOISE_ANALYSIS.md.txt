    42	## Suppression Rules
    43	
    44	### Rule 1: Priority-based filtering (always active)
    45	
    46	Suppress events with `priority == LOW` or `priority == BACKGROUND` unless explicitly subscribed.
    47	Evidence: `[Direct code] nl_...event_coordinator.py.txt L48-L55` (EventPriority: CRITICAL=1, HIGH=2, MEDIUM=3, LOW=4, BACKGROUND=5).
    48	
    49	### Rule 2: Deep-focus gating (when `current_focus_mode == "deep"`)
    50	
    51	Pass only CRITICAL and HIGH events. Block non-critical interruptions.
    52	Evidence: `[Direct code] nl_...event_coordinator.py.txt L331-L338`.
    53	
    54	### Rule 3: Energy-level gating (when `current_energy_level == "low"`)
    55	
    56	Suppress events with `cognitive_load > 0.7` to prevent overwhelm during low-energy states.
    57	Evidence: `[Direct code] nl_...event_coordinator.py.txt L340-L344`.
    58	
    59	### Rule 4: Event flood protection (always active)
    60	
    61	When >10 similar events fire within one minute, log flood warning and suppress duplicates.
    62	Evidence: `[Direct code] nl_...event_coordinator.py.txt L346-L349` (flood detection threshold).
    63	
    64	### Rule 5: Batch-eligible event coalescing
    65	
    66	Events marked `can_batch=True` in ADHDMetadata should be coalesced into periodic digests rather than delivered individually. Events with `time_sensitive=True` bypass batching.
    67	Evidence: `[Direct code] nl_...event_bus.py.txt L34-L38` (ADHDMetadata: interruption_allowed, focus_required, time_sensitive, can_batch).
    68	
    69	### Rule 6: Evidence audit noise suppression
    70	
    71	During PM evidence triage, suppress binary-cache grep rows and non-PM status matches unless running an explicit cross-plane audit.
    72	Evidence: `[Scan hit] 30_memory_burden_search.txt L2,L9-L11`; `20_friction_search.txt L81-L96`.
    73	
    74	### Rule 7: Citation integrity gate
    75	
    76	Any event classification without a verifiable evidence citation is invalid and must be moved to the Open Questions section as UNKNOWN.

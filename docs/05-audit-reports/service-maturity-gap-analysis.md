# Service Maturity & Gap Analysis Report

**Date**: 2026-02-06
**Scope**: 30+ Services in Dopemux Ecosystem
**Auditor**: Deep Dive Agent

## 1. Executive Summary

A comprehensive audit of the Dopemux "Two-Plane Architecture" reveals a highly modular system with robust code-level implementation but historical gaps in documentation and registry completeness (now largely remediated).

**Key Findings:**
*   **Registry Coverage**: Improved from ~50% to ~95% after this audit.
*   **Documentation**: Standardized via "Master History" artifacts for all 16 service groups.
*   **Testing**: High variability. Core services (`conport`, `adhd-engine`) have strong coverage; peripheral services (`voice-commands`, `leantime-bridge`) are largely untested prototypes.

## 3. Critical Gaps & Technical Debt

### 🛑 High Priority (P0 - Security & Stability)
1.  **Testing Vacuums**: `workspace-watcher` and `activity-capture` are running in production but have *zero* visible test coverage.
2.  **Hardcoded Ports**: `voice-commands` uses `3007` but this was absent from the registry until today.
3.  **Library vs Service Ambiguity**: `serena` and `dope-context` are critical but treated as libraries. They should be formalized or strictly defined as shared libs.

### ⚠️ Medium Priority (P1 - Maintainability)
1.  **Duplicate Functionality**: `dddpg` and `dope-query` overlap significantly. `dope-query` claims to be the MCP server for `dddpg`, but clarify boundaries.
2.  **Orphaned Integrations**: `slack-integration` is a loose script (`notifier.py`). It should be containerized or folded into `adhd-notifier`.

### 📉 Low Priority (P2 - Optimization)
1.  **LiteLLM Config**: Configuration is static. Should move to dynamic config via `dope-config` to allow runtime model switching.
2.  **Monitoring Disconnect**: `monitoring` stack exists but doesn't scrape `adhd-dashboard` metrics automatically yet.

---

## 4. Recommendations

1.  **Containerize Scripts**: Move `slack-integration` into `adhd-notifier` as a channel module.
2.  **Formalize Libraries**: Create a specific `libs/` folder for `dope-context` and `serena-core` to distinguish them from `services/`.
3.  **Test Initiative**: Launch a "Test The Periphery" sprint to add basic smoke tests to `workspace-watcher`.

# Log Task Packets for Rapid Debugging

Use task packets when you need a full multi-service debug bundle you can hand to another engineer (or another agent) without losing context.

## What Gets Collected

`scripts/collect_task_packet.py` captures:

1. **Compose-level state**
- `docker compose ps`, image list, service list
- per-service logs with timestamps and time window controls

2. **Service-level diagnostics**
- `/health` snapshots from `services/registry.yaml`
- `/metrics` snapshots when available

3. **Component-level logs**
- local files from `./logs/` (tail-captured)
- normalized JSONL logs per service for programmatic analysis

4. **Investigation artifacts**
- `summary.json` (machine-readable report)
- `README.md` (human triage summary)
- `TASK_PACKET.md` (handoff prompt template)
- `analysis/packet_metrics.prom` (Prometheus exposition)
- `analysis/datadog_series.json` (Datadog series payload)

## Quick Start

```bash
python3 scripts/collect_task_packet.py --task-id bridge-latency --since 90m --services all
```

Or through Make:

```bash
make task-packet TASK_ID=bridge-latency SINCE=90m SERVICES=all
```

Output is written under:

`reports/task-packets/<task-id>-<timestamp>/`

An archive is generated by default:

`reports/task-packets/<task-id>-<timestamp>.tar.gz`

## Multi-Mode Consumption

Use whichever mode fits the debugging workflow:

1. **Human triage**
- open `README.md`
- open `TASK_PACKET.md`

2. **CLI grep/ripgrep workflow**

```bash
rg -n "error|critical|exception|traceback" reports/task-packets/<packet>/logs
```

3. **Structured processing**
- `normalized/*.jsonl` for scripts, notebooks, LLM pipelines
- `summary.json` for dashboards and ticket enrichment

4. **Metrics systems**
- Prometheus: ingest `analysis/packet_metrics.prom`
- Datadog: use `analysis/datadog_series.json`

## Datadog Emission (Optional)

If `DD_API_KEY` is available:

```bash
python3 scripts/collect_task_packet.py \
  --task-id incident-123 \
  --emit-datadog \
  --datadog-site datadoghq.com
```

The script writes `analysis/datadog_emit_result.json` with API response details.

## Recommended Investigation Flow

1. Generate packet from the failing environment.
2. Attach the `.tar.gz` to the task/incident.
3. Start with `summary.json` to identify highest-error services.
4. Correlate `logs/` with `health/` and `metrics/`.
5. Record root cause + fix plan back into task tracking.

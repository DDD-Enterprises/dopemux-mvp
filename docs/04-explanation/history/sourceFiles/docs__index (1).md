## Key Features Summary

### 🧠 ADHD Accommodations
- **Context Preservation** - Auto-save every 30s, <500ms restore
- **Attention Adaptation** - Real-time cognitive state classification
- **Task Decomposition** - 25-minute focused chunks with progress tracking
- **Gentle Guidance** - Supportive language, clear next steps
- **Decision Reduction** - Maximum 3 options to prevent overwhelm

### 🤖 AI Integration
- **Claude Code Integration** - Seamless launch with MCP servers
- **Multi-Model Support** - Latest models (Opus 4.1, Sonnet 4, GPT-5, O3-Pro)
- **Attention-Based Routing** - Different models for different cognitive states
- **Hybrid Search** - Milvus + BM25 for context-aware assistance

### 🔧 Developer Experience
- **Rich CLI** - Beautiful terminal interface with progress visualization
- **Multiple Formats** - YAML, TOML, JSON configuration support
- **Template System** - Pre-configured setups for Python, JS, Rust, etc.
- **Shell Integration** - Bash/Zsh completion and helpful aliases

## Contributing to Documentation

### ADHD Documentation Principles

When contributing to Dopemux documentation:

1. **Start with the outcome** - What will the user achieve?
2. **Use active voice** - "Click the button" not "The button should be clicked"
3. **Provide context** - Why is this important for ADHD developers?
4. **Include examples** - Show, don't just tell
5. **Be encouraging** - Use positive, supportive language

### Documentation Standards

```markdown
## Standard Section Structure

### Overview
Brief description of what this section covers

### Quick Start
For users who need immediate results

### Detailed Guide
Step-by-step instructions with examples

### Troubleshooting

### Advanced Topics
For power users and customization
```

Our excavation of the git history revealed two major "Lost Civilizations" - sophisticated systems that were built and then deleted or abandoned. These losses explain part of the roadmap friction, but they are not the only reason current features are incomplete.

**Key Findings:**

## 2. Excavation Site A: The "Zen" Server

**Location**: `docker/mcp-servers/zen/zen-mcp-server`
**Status**: **DELETED in active tree, PARTIALLY REPLACED**

**What it was**:
"Zen" appears to have been the original "Super Brain" of the project. The deletion logs show python files for:
- `thinkdeep.py`: Likely Chain-of-Thought reasoning.
- `planner.py`: Automated task decomposition.
- `refactor.py`: Code improvement logic.
- `secaudit.py`: Security scanning.

**Impact of Loss**:
The `services/agents/README.md` explicitly references wrapping "Zen" tools.
> "TaskDecomposer: 90% Zen/planner wrapper"
> "ToolOrchestrator: 80% Zen/listmodels wrapper"

**Conclusion**: The Zen deletion is material, but the "7 Infrastructure Agents" are not simply vaporware from this cause alone; several agent modules now exist and require hardening, integration, and parity work.

## 3. Excavation Site B: The Codex of Personas

**Location**: `.claude/commands/`
**Status**: **DELETED**

**What it was**:
A structured library of "Meta-Prompts" that allowed the user to invoke specific AI behaviors.
- `dx/assess.md`: "Assess the situation"
- `dx/prd-parse.md`: "Read this PRD"
- `rfc-lint.md`: "Lint this RFC"
- `doc-new.md`: "Write documentation"

**Impact of Loss**:
We moved from "Explicit Command Files" to "Implicit Agent Behavior". While the move to agents is architecturally sound, we lost the *determinism* and *specificity* of these hand-crafted prompt files.

## 4. Recommendations

1. **Resurrect Zen**: Check if `zen-mcp-server` exists in another repo or needs to be rebuilt from the `__pycache__` ghosts (unlikely) or re-implemented. The Agents generally cannot exist without it.
1. **Restore Critical Prompts**: The logic in `prd-parse.md` and `rfc-lint.md` likely contained valuable heuristic rules. We should dig them out of git history if we want to teach the new Agents how to do these tasks effectively.
1. **Document Zen -> PAL parity map**: Track which Zen-era capabilities are implemented, partially implemented, or planned in PAL-era services.

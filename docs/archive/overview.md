# Documentation Archive

This directory contains historical documentation, completed project records, and deprecated files.

## Structure

### `session-notes/`
Sprint summaries, session notes, and daily progress logs organized by date.

**2025-10/**
- ADHD Dashboard development sessions
- ConPort integration milestones
- Monitoring design sprints
- Security audit sessions
- Various completion summaries

### `completed-projects/`
Documentation from completed initiatives, kept for historical reference:

- **ConPort Projects:**
  - Event Bridge Success
  - HTTP Migration
  - LSP Hover Feature
  - Production Deployment

- **Integration Projects:**
  - Leantime MCP Integration
  - MetaMCP Complete Setup

- **Infrastructure:**
  - Multi-Instance Workspace Fix
  - Docker Cleanup
  - Security Audit & Fixes
  - Token Limit Fixes

- **Dashboard:**
  - Compact Dashboard Implementation
  - Orchestrator Integration

### `deprecated/`
Outdated or superseded documentation:

- Old action plans
- Superseded quick start guides
- Deprecated monitoring indexes
- Historical changelogs

## Usage Guidelines

### When to Archive

Archive documentation when:
- A project/feature is complete and stable
- Session/sprint notes are from previous months
- Documentation has been superseded by newer versions
- Files are no longer actively referenced

### When NOT to Archive

Keep in active docs if:
- Still under active development
- Referenced in current work
- Part of core documentation
- Needed for ongoing operations

## Finding Archived Content

Use the search function or browse by:
1. **Date**: `session-notes/YYYY-MM/`
2. **Project**: Look for related file names in `completed-projects/`
3. **Type**: Check `deprecated/` for old versions

## Retention Policy

- **Session Notes**: Keep indefinitely for project history
- **Completed Projects**: Keep for at least 1 year
- **Deprecated Docs**: Review annually, remove if no longer useful
